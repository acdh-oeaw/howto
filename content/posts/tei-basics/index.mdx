## TEI Makrostruktur

### Grundlagen

Bevor wir uns an die Arbeit mit dem spezifischen Beispiel machen, gilt es den Aufbau eines validen TEI-Dokuments zu verstehen. Der erste Schritt ist hierbei das Verständnis der TEI-Makrostruktur, die die übergeordneten Strukturen eines TEI-konformen Dokuments bzw. eines TEI-Korpus beschreibt. Diese Struktur umfasst und gliedert sowohl die Metadaten über das Dokument wie auch die annotierten Inhalte.

### Praktische Anwendung

Ein **TEI-konformes Dokument** wird vom TEI-Element `<TEI>`  - dem Wurzelelement - umschlossen. Dass Attribut **'XML Namespace'** `xmlns`, in dem es auf eine eindeutige Definition der Elemente in einem XML-Schema referenziert wird, die innerhalb des spezifischen Dokuments verwendet wird.

Eine TEI-Datei gliedert sich grundsätzlich in den **'TEI-Header'** `<teiHeader>` **(*Sprungmarke zu TEI Header Kapitel*)**, der die Metadaten zum elektronischen Dokument enthält, sowie den annotierten **Text**, der vom Element `<text>` umschlossen wird.

```xml
<TEI xmlns="http://www.tei-c.org/ns/1.0">
	<teiHeader>
		<fileDesc>
			<titleStmt> … </titleStmt>
			<sourceDesc> … </sourceDesc>
			<publicationStmt> … </publicationStmt>
		</fileDesc>
	</teiHeader>
	<text>
		<body>
			<p>Ein Absatz.</p>
		</body>
	</text>
</TEI>
```

Ein Text kann entweder 'unitary' - also ein organisches Ganzes – oder 'composite' sein – also aus mehreren voneinander unabhängigen Komponenten bestehend. Ein Text kann optional einen **'front matter'** `<front>` oder auf Deutsch **Titelei** enthalten, die beispielsweise die Inhalte der **Titelseite** `<titlePage>` enthalten kann, wie auch die Position einer automatisch generierten Gliederung wie einem Index, einem Inhaltsverzeichnis o.ä.. Für Letzteres kommt das Element **'text division'** `<divGen>` mit einer der folgenden type-Attribute zum Einsatz:

* `index`: hier soll ein automatisch generierter Index eingesetzt werden.
* `toc`: Inhaltsverzeichnis
* `figlist`: Abbildungsliste
* `tablist`: Tabellenliste

Der erforderliche **'text body'** `<body>` umschließt den Hauptext eines einzelnen 'unitary' Dokuments.

Das optionale Element **'back matter'** `<back>` enthält den Anhangteil eines Dokuments, und kann z.B. **Index-Einträge** `<index>` enthalten.

Im folgenden Code-Beispiel sieht man die Makrostruktur eines 'unitary'-Textes, z.B. eine Monografie oder ein Drama, das eine Titelseite und ein Inhaltsverzeichnis enthält, und zudem einen Index am Ende des Buches.

```xml
…
<text>
	<front>
		<titlePage>…</titlePage>
		<divGen type="toc"/>
	</front>
	<body>
		<p>Mein erster Absatz.</p>
	</body>
	<back>
		<index>…</index>
	</back>
</text>
…
```

Wurde im obenstehenden Beispiel ein 'unitary'-Text beschrieben, so zeigt das untenstehende Beispiel, wie man einen 'composite'-Text annotiert, beispielsweise einen Jahrgang einer Zeitschrift mit unterschiedlichen Ausgaben, wie es bei der Zeitschrift aus unserem Beispiel der Fall sein könnte. Hierfür kommt das `<group>`-Element zum Einsatz, das eine Reihe an zusammengehörigen Texten `<text>` beinhaltet.

Man sieht in diesem Beispiel das jeder Text in einer Gruppe mit dem Attribut `xml:id` eine im Gesamtdokument einzigartige Identifikation hat. Wichtig ist, dass diese ID immer mit einem Buchstaben oder einem Unterstrich beginnen muss und ansonsten nur Buchstaben, Zahlen, Bindestriche, Unterstriche, oder Punkte enthalten darf, um als valides XML zu gelten. [Mehr Informationen zu xml:id](https://teibyexample.org/tutorials/TBED01v00.htm?target=globalattributes#xmlid).

```xml
<TEI xmlns="http://www.tei-c.org/ns/1.0">
	…
	<text>
		<front>
			<!-- Titelei für den gesamten Jahrgang -->
		</front>
		<group>
			<text xml:id="fw1920-01">
				<!-- Ausgabe 1 -->
			<text>
			<text xml:id="fw1920-02">
				<!-- Ausgabe 2 -->
			<text>
		</group>
	</text>
</TEI>
```

<SideNote type="note" title="Merke">
Die Grenze zwischen 'unitary'- und 'composite'-Text kann fließend sein. Eine Essay-Sammlung ist ein möglicher Grenzfall, bei dem je nach Fragestellung an den Text als eine Einheit oder aber als eine Reihe von in sich geschlossenen Elementen betrachtet werden kann. Hier muss die Person, die die Texte kodiert, entscheiden, ob sie die Texte als 'unitary' oder 'composite' kodieren möchte.

*Beispiel Fragestellungen für unitary oder composite ergänzen*
</SideNote>

Eine Reihe von eigenständigen TEI-Elementen `<TEI>` mit jeweils Daten und Metadaten zu einem einzelnen Dokument können zu einem **Korpus** `<teiCorpus>` mit einem eigenen Header und den zugehörigen Korpus-Metadaten zusammengefügt werden. Jedes TEI-Element könnte hierbei beispielsweise eine Sammlung an kodierten Texten, eine Version eines Textes, oder Samples eines Sprachkorpus u.a. enthalten.

```xml
<teiCorpus xmlns="http://www.tei-c.org/ns/1.0">
	<teiHeader>
			<!-- Metadaten für das Corpus-->
	</teiHeader>
	<TEI xml:id="t001">
		<teiHeader>
			<!-- Metadaten für Text 1 -->
		</teiHeader>
		<text>
			<!-- Text 1 -->
		</text>
	</TEI>
	<TEI xml:id="t002">
		…
	</TEI>
	<TEI xml:id="t003">
		…
	</TEI>
</teiCorpus>
```
